Gallica Grapher vise à rendre l'exploration de milliers de périodiques facile, agréable et centrée sur l'humain. En cliquant sur le graphique, vous obtenez un échantillon d'occurrences de cette période.

Tout est en cours de développement! Si vous avez une suggestion ou un problème, vous pouvez me contacter en DM sur Twitter.

Mes remerciements aux projets suivants:

1. Les Archives Nationales Françaises [Gallica API](https://api.bnf.fr/fr/api-document-de-gallica), pour une API ouverte et riche en données;
2. [Gallicagram](https://shiny.ens-paris-saclay.fr/app/gallicagram), pour leurs efforts considérables dans la numérisation de l'archive Gallica et la construction d'un vaste ensemble de données normalisées sur le nombre de mots, accessibles via une API rapide. Cette application ne serait pas possible autrement. Pour une analyse lexicale plus approfondie, essayez leur application!
3. L'outil Javascript [React](https://react.dev/), qui a rendu la construction d'interfaces réactives un plaisir.

#### De nouveaux contributeurs sont les bienvenus

N'hésitez pas à cloner le code ou créer un nouveau problème sur [Github](https://github.com/gleasonw/gallica-grapher). J'adorerais que cet outil devienne un projet communautaire, maintenu et conçu par son public. Le code d'analyse de base est écrit en Python, le frontend en Typescript et React. Je travaille pour rendre le code plus accessible et lisible.

N'ayez pas peur de jouer avec le code -- l'écosystème du web a énormément progressé au cours des dix dernières années environ. Je ne connaissais rien à la programmation d'une application réactive lorsque j'ai commencé ce projet à l'été 2022.
